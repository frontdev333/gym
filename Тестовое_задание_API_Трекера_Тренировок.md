# Тестовое задание: API Трекера Тренировок

Необходимо разработать небольшой backend сервис для учёта выполненных упражнений и получения агрегированной статистики.

## Сценарий

Пользователь выполняет различные упражнения.

Например:

* Жим лёжа
* Приседания
* Подтягивания
* Планка
* Бег

Сервис должен позволять:

1. Создавать упражнения.
2. Фиксировать факт выполнения упражнения пользователем.
3. Получать статистику по выполненным упражнениям.

Способ организации API кандидат определяет самостоятельно.

## Функциональные требования

### Упражнения

Пользователь должен иметь возможность создавать упражнения.

Минимально необходимо хранить название упражнения.

### Выполнения упражнений

Пользователь должен иметь возможность зафиксировать выполнение упражнения.

Необходимо хранить:

* пользователя;
* упражнение;
* дату и время выполнения.

Состав дополнительных полей кандидат определяет самостоятельно.

### Статистика

Сервис должен предоставлять агрегированную статистику по пользователю.

Статистика должна содержать:

1. Общее количество выполнений упражнений.
2. Количество выполнений за текущий день.
3. Количество выполнений за последние 7 дней с группировкой по дням.

Формат ответа кандидат определяет самостоятельно.

## Технические требования

Необходимо использовать:

1. Go.
2. PostgreSQL.
3. Docker и Docker Compose.

## Требования к реализации

1. Использовать миграции базы данных.
2. Реализовать базовую валидацию входных данных.
3. Обрабатывать ошибки и возвращать корректные HTTP статусы.
4. Структурировать проект таким образом, чтобы код было удобно поддерживать и расширять.
5. Агрегации должны выполняться на стороне PostgreSQL.

Допускается использование любых библиотек и фреймворков, если их выбор обоснован.

## Репозиторий

Результат необходимо разместить в Git репозитории.

В репозитории должна присутствовать инструкция по запуску проекта.

README должен содержать:

1. Требования для запуска.
2. Команды для сборки и запуска.
3. Команды для применения миграций.
4. Примеры использования API.
5. Краткое описание структуры проекта.
6. Краткое описание принятых технических решений.

## Что ожидается на выходе

1. Ссылка на Git репозиторий.
2. Рабочее приложение, запускаемое через Docker Compose.
3. Миграции базы данных.
4. README с инструкцией по запуску.

## Что не требуется

Для выполнения задания не требуется:

1. Реализовывать авторизацию и аутентификацию.
2. Разрабатывать frontend.
3. Добавлять очереди сообщений.
4. Настраивать кеширование.
5. Реализовывать сложную бизнес-логику тренировок.

Основная цель задания - оценить структуру проекта, работу с PostgreSQL, реализацию агрегирующих запросов и общий подход к разработке сервиса.
